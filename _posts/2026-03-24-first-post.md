很多人搭博客时，第一反应是功能越多越好。但如果目标是长期写作，真正重要的是：**结构稳定、修改成本低、部署路径简单**。

<!--more-->

这次我选择 Jekyll + GitHub Pages，主要有三个原因：

1. 与 GitHub Pages 原生兼容，部署几乎零运维。
2. Markdown 写作体验稳定，迁移成本可控。
3. 纯静态输出，不依赖数据库和后端接口。

## 我最看重的工程约束

- 文件结构清晰：布局、样式、脚本、内容分层。
- 功能够用即可：归档、分类、标签、搜索、RSS。
- 可预测行为：少引入复杂依赖，避免后续维护负担。

## 一个示例代码块

```bash
bundle exec jekyll serve
```

本地启动后，优先检查移动端排版、代码块可读性和文章页上下篇跳转。

## 结语

博客是长期项目，不必一开始做重。先把写作路径跑通，再逐步迭代，是更稳妥的做法。
